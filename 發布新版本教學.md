# 🚀 CYT PDF 工具：發布新版本教學

本專案已整合自動化發布流程。只需執行腳本，即可完成版本更新、打包 EXE、上傳 GitHub 等繁瑣步驟。

---

## 🏗️ 發布前的準備
1.  **安裝工具**：確保電腦已安裝 [GitHub CLI (`gh`)](https://cli.github.com/)。
2.  **登入狀態**：在終端機執行 `gh auth login` 確保已登入 GitHub 帳號。
3.  **內容檢查**：確認 `使用說明.md` 內容為最新。

---

## 🛠️ 發布流程 (一鍵完成)

### 步驟 1：雙擊執行
雙擊專案根目錄下的 **`一鍵發布新版本.bat`**。

### 步驟 2：輸入新版本號
腳本會自動偵測 `app.py` 中的目前版本。
- 直接按 **Enter**：使用建議的下一個版本號 (例如 1.0.0 -> 1.0.1)。
- 手動輸入：例如輸入 `1.1.0`。

### 步驟 3：輸入更新日誌
簡短描述本次更新的重點，這會顯示在 GitHub Release 的說明欄。

### 步驟 4：自動化執行
腳本將依序完成：
- ✅ 修改 `app.py` 內的版本號。
- ✅ 修改 `使用說明.md` 內的版本號。
- ✅ 使用 PyInstaller 打包成 `CYT_PDF_Tool.exe`。
- ✅ 將 `使用說明.md` 轉成 `程式說明.txt`。
- ✅ 建立 ZIP 壓縮包。
- ✅ 執行 `git push` 同步代碼。
- ✅ 建立 GitHub Release 並上傳檔案。

---

## 🔄 平時開發：多端同步教學
若您在多台電腦 (電腦 A 與 電腦 B) 之間開發，請善用進度同步腳本：

### 在電腦 A (開發結束時)
1. 雙擊 **`上傳進度.bat`**。
2. 這會自動執行 `git add / commit / push`，將最新進度推送到雲端。

### 在電腦 B (開始開發前)
1. 雙擊 **`下載進度.bat`**。
2. 這會自動執行 `git pull`，將電腦 A 的進度下載到本機。

---
*💡 小提示：若自動化流程失敗，請檢查網路連線或 `gh auth status`。*
