# 🚀 專案自動化功能移植指南 (AI 協作模版)

本資料夾包含了由 AI 協作開發的「自動化發布與同步套件」。當您開啟新專案時，只需將本資料夾內的檔案複製到新專案根目錄，並完成簡單設定，即可擁有專業級的開發工作流。

---

## 📦 組件清單

1.  **`release_helper.py`**：核心發布助手。處理版本同步、MD 轉 TXT、PyInstaller 打包、Git 提交與 GitHub Release 上傳。
2.  **`一鍵發布新版本.bat`**：雙擊即可啟動發布流程。
3.  **`上傳進度.bat`**：快速執行 `git add / commit / push`，適合在下班前同步程式碼。
4.  **`下載進度.bat`**：快速執行 `git pull`，適合在上班前同步其他電腦的進度。
5.  **`使用說明.md`**：軟體說明書模版（發布時會自動轉成 `程式說明.txt`）。
6.  **`優化方向.md`**：專案開發進度與藍圖管理模版。
7.  **`icon.ico`**：預設圖示檔案（打包 EXE 時必備）。

---

## 🛠️ 新專案移植步驟

### 第一步：環境準備
1.  確保電腦已安裝 **Git** 與 **GitHub CLI (`gh`)**。
2.  在新專案資料夾內執行一次：
    ```bash
    git init
    gh auth login  # 若尚未登入
    gh repo create [新專案名稱] --public
    ```

### 第二步：修改設定 (重要)
1.  **修改主程式名**：
    打開 `release_helper.py`，搜尋 `main.py` 並將其改為您新專案的進入點檔名（例如 `app.py`）。
2.  **定義版本變數**：
    在您的新主程式頂部，務必定義以下兩個變數：
    ```python
    APP_VERSION = "1.0.0"
    GITHUB_REPO = "您的帳號/新專案名稱"
    ```

### 第三步：開始使用
*   **平時開發**：利用 `上傳進度.bat` 與 `下載進度.bat` 進行多端同步。
*   **準備發布**：更新 `使用說明.md` 內容後，雙擊 `一鍵發布新版本.bat`。
