# 專案初始化檢查流程

執行以下步驟來確認專案中的程式碼並生成功能摘要報告：

## 1. 探索專案結構
- 列出專案根目錄的內容
- 識別主要的資料夾和檔案
- 找出設定檔（如 package.json, requirements.txt, pom.xml 等）

## 2. 分析程式碼組織
- 確認主要的原始碼目錄（src, app, lib 等）
- 檢查測試檔案位置
- 查看文件目錄（docs, README 等）

## 3. 檢查核心功能
- 閱讀主要入口檔案
- 分析關鍵模組和類別
- 理解資料流和架構模式

## 4. 產生功能摘要報告
- 列出專案類型和使用的技術棧
- 描述主要功能模組
- 說明專案架構和組織方式
- 標註重要的設定和相依性
- 提供程式碼品質觀察（如有）
